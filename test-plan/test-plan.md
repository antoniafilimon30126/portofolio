# Test Plan – SauceDemo Web Application

## 1. Project Overview

This test plan describes the testing strategy for the SauceDemo web application.

The purpose of testing is to validate the core functionalities of the application that impact the user experience.

The testing focuses on the following areas:

- user authentication and login validation
- product management in the shopping cart
- checkout process and validation of required fields
- navigation within the application
- product sorting and display

---

## 2. Scope and Objectives

The objective of this testing process is to verify that the main functionalities of the application work correctly and meet the expected requirements.

Main objectives:

- verify correct login functionality
- validate adding and removing products from the cart
- confirm checkout behavior for valid and invalid scenarios
- verify navigation elements and user interactions
- identify defects that may affect user experience

---

## 3. Types of Testing

The following testing types were used:

- Functional Testing
- Black-box Testing
- UI Testing
- Manual Testing

---

## 4. Test Strategy

Testing is performed using a black-box approach, focusing on validating application behavior from the user's perspective.

Both positive and negative scenarios are covered.

The testing process includes:

1. analysis of application functionality
2. design of test cases
3. execution of tests
4. documentation of results
5. reporting detected defects

---

## 5. Features to be Tested

The following features are included in testing:

- Login functionality
- Product catalog
- Add product to cart
- Remove product from cart
- Checkout process
- Product sorting
- Navigation menu
- Logout functionality

---

## 6. Test Environment

Testing was performed in the following environment:

Operating System: macOS 15.7.4  
Browser: Google Chrome  
Application URL: https://www.saucedemo.com

---

## 7. Entry Criteria

Testing can start when:

- the application is accessible
- the testing environment is properly configured
- test data is available

---

## 8. Exit Criteria

Testing is considered completed when:

- all test cases are executed
- test results are documented
- identified defects are reported

---

## 9. Risks and Limitations

Potential risks during testing:

- possible updates to the SauceDemo application
- dependency on internet connection
- testing limited to a single browser
